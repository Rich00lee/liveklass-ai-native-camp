> 워크숍 마무리 + 다음 스텝 설계

## EXPLAIN

### 팀 확산 전략

워크숍이 끝나는 게 아니라 **시작**이다.

**3단계 확산 로드맵**:

**1단계 (이번 주): 개인 정착**
- 매일 Claude Code 최소 1번 사용
- 오늘 만든 Skill 실제 업무에 적용
- 막히면 바로 Claude에게 물어보기 (구글링 전에 Claude 먼저)

**2단계 (다음 달): 팀 공유**
- 팀 내 Slack 채널 만들기: `#ai-tips` — Claude Code 활용 팁 공유
- 팀원 중 가장 활용 잘 하는 사람 → "AI 앰버서더" 역할
- 매월 팀 미팅에서 "이번 달 베스트 자동화" 공유

**3단계 (분기): 조직 확산**
- 이 워크숍 자료를 신규 입사자 온보딩에 활용
- 다른 팀에 워크숍 공유 (CS팀, 마케팅팀 등)
- 성과 측정: 절약된 시간, 처리량 증가, 오류 감소

**앰버서더 역할**:
워크숍에 참가한 개발자 팀원에게:
- 팀원 기술 질문 1차 답변
- mcp.json 설정 도움
- 새로운 MCP 서버 발굴 + 공유

### 개인 다음 스텝

각자가 "다음 주에 꼭 해볼 것" 1가지를 정한다.

예시:
- "매일 아침 `/slack-brief` 실행하기"
- "파이프라인 시트 분석을 매주 금요일 Claude로 하기"
- "CS 문의 10건 중 3건은 `/cs-reply`로 초안 만들기"

```
다음 한 달 동안 Claude Code를 업무에 정착시킬 구체적인 액션 플랜을 만들어줘.
주차별로, 실천 가능한 수준으로 작성해줘.
```
