## EXPLAIN

### 1분 데모 피칭 준비

발표는 기술 설명이 아니다. **"이게 내 업무를 어떻게 바꿨는지"** 를 보여주는 것.

**1분 피칭 구조**:

    1. 문제 (10초): "저는 매일 OOO을 하는 데 OO분이 걸렸어요."
    2. 해결 (10초): "이제 Claude에게 /[skill-name]을 치면"
    3. 시연 (30초): 실제로 실행해서 결과 보여주기
    4. 임팩트 (10초): "이걸로 하루에 약 OO분이 절약됩니다."

**데모 팁**:
- 실제 업무 데이터로 시연 (가장 설득력 있음)
- 결과가 나오는 속도 자체가 임팩트 — 시간 재기
- 완벽할 필요 없음 — "아직 개선 중인데" 도 OK

**예시 피칭**:
> "저는 매일 아침 Slack을 정리하는 데 20분이 걸렸어요.
> 이제 `/slack-brief`를 치면 — (실행) — 30초 만에 오늘의 핵심 5가지가 나와요.
> 하루 약 18분, 월 6시간을 절약할 수 있어요."

## EXECUTE

Claude에게 데모 준비를 도움받으세요:

Claude Code에 입력:

    나는 라이브클래스 [직군]이야.
    Session 3에서 [만든 Skill 이름]을 만들었어.
    이걸 팀에 1분 안에 발표해야 하는데, 피칭 스크립트 작성해줘.

    포함할 내용:
    - 기존에 수동으로 하던 방법과 시간
    - 새 방법 (Skill 실행)
    - 절약 시간 또는 품질 향상

발표 준비가 끝났다면, 만든 Skill을 GitHub에 올려보세요 (숙제):

    # 숙제: Session 5 발표 전까지 GitHub에 올리기
    # 1. GitHub에 리포 만들기 (또는 기존 리포에 추가)
    # 2. .claude/skills/[내 skill]/ 폴더를 push
    # 3. README에 사용법 간단히 적기
