## EXPLAIN

### SHOWCASE.md — 발표 자료 템플릿

Session 5 사내 공유회에서 발표할 내용을 미리 정리하는 문서다.

**템플릿 구조** (6개 섹션):

| 섹션 | 질문 | 예시 |
|------|------|------|
| 1. 해결하고 싶은 문제 | 어떤 업무가 불편했나? | "CS 문의 응답에 매번 30분씩 걸린다" |
| 2. 기존 방식의 병목 | 어디서 막혔나? | "매번 비슷한 내용인데 처음부터 작성한다" |
| 3. 지향했던 방향성 | AI로 어떻게 바꾸고 싶었나? | "자주 오는 유형은 초안을 자동 생성하고 싶다" |
| 4. 현재 구현 단계 | 지금까지 만든 것은? | "cs-auto-reply 스킬 완성, 5개 유형 대응" |
| 5. 추후 과제 | 더 하고 싶은 것은? | "Slack 알림 연동, 응답 품질 평가" |
| 6. 배운 점 / 소감 | 캠프에서 얻은 것은? | "Claude에게 시키는 법을 배웠다" |

### 마크다운 기본 문법 복습

```markdown
# 제목 (큰 글씨)
## 소제목
- 목록 항목
- **굵게** 표시
> 인용문 (강조하고 싶은 문장)
```

> 💡 마크다운을 모르더라도 걱정 마세요. Claude에게 "이 내용을 예쁘게 정리해줘"라고 하면 됩니다.

### 좋은 발표 자료의 포인트

1. **구체적 숫자**: "시간이 줄었다" → "30분 → 5분으로 줄었다"
2. **전후 비교**: 기존 방식 vs AI 방식을 나란히 보여주기
3. **실제 실행 결과**: 스크린샷이나 실행 결과를 넣으면 임팩트 UP

## EXECUTE

Claude에게 아래 순서대로 시켜보세요:

**1단계: 내 폴더 만들기 + 템플릿 복사**
```
"lk-ai-camp-showcase 레포에서 submissions/내이름 폴더를 만들고,
template/SHOWCASE.md를 거기로 복사해줘"
```

**2단계: SHOWCASE.md 작성하기**

두 가지 방법 중 편한 걸 선택하세요:

**방법 A — Claude와 대화하면서 작성**
```
"내 SHOWCASE.md를 같이 채워줘.
나는 [직군]이고, 캠프에서 [만든 것]을 했어.
질문하면서 하나씩 채워줘"
```

**방법 B — 직접 내용을 말하고 Claude가 정리**
```
"내 SHOWCASE.md에 이 내용을 넣어줘:
- 문제: [설명]
- 기존 방식: [설명]
- AI로 한 것: [설명]
- 결과: [설명]"
```

> 💡 스킬이나 에이전트를 만들었다면, 해당 파일도 같은 폴더에 복사해 넣으세요.
> Claude에게 "내가 만든 스킬 파일을 submissions/내이름/ 폴더에 복사해줘"라고 하면 됩니다.

**3단계: 작성 결과 확인**
```
"submissions/내이름/SHOWCASE.md 보여줘"
```

## QUIZ

```json
AskUserQuestion({
  "questions": [{
    "question": "SHOWCASE.md의 6개 섹션 중, 발표에서 가장 임팩트를 주는 건 뭘까요?",
    "header": "Block 1 퀴즈",
    "options": [
      {"label": "A. 해결하고 싶은 문제"},
      {"label": "B. 기존 방식의 병목"},
      {"label": "C. 현재 구현 단계 (실제 결과물)"},
      {"label": "D. 배운 점 / 소감"}
    ],
    "multiSelect": false
  }]
})
```

정답: **C**
- 물론 모든 섹션이 중요하지만, 청중이 가장 관심 있는 건 **"실제로 뭘 만들었고 어떤 결과가 나왔는지"**
- 스크린샷, 실행 결과, 전후 비교가 있으면 발표의 설득력이 크게 올라간다
- 다만 "왜 이 문제를 풀려고 했는지(섹션 1-2)"가 먼저 공감을 만들어야 효과적이다
