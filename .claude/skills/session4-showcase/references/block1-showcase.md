> 데모 발표 세션 — 각자 만든 Skill을 팀에 공유한다

## EXPLAIN

### 팀 데모 발표

각자 만든 자동화를 팀에게 발표하는 시간.

**진행 방식**:
- 참가자 1명당 1분 데모 (최대 5명 × 1분 = 5분)
- 발표 후 팀원들이 바로 설치하거나 질문
- 가장 많은 관심을 받은 Skill → 팀 공식 Skill로 채택

**발표 체크리스트**:
- [ ] 발표할 Skill이 실행 가능한 상태인가?
- [ ] 실제 업무 데이터로 데모할 준비가 됐는가?
- [ ] 1분 피칭 스크립트를 연습했는가?
- [ ] 팀원이 설치하려면 어떻게 하면 되는지 알고 있는가?

**Skill 공유 방법**:
```bash
# 방법 1: SKILL.md 파일을 Notion/Slack에 공유
# 방법 2: GitHub 레포에 올린 후 npx skills add 명령어 공유
```

## EXECUTE

지금 발표 리허설을 Claude와 해보세요:

```
나 대신 팀원 역할을 해줘.
내가 아래 Skill을 발표할 테니까, 발표 후에 팀원 입장에서 질문해줘.

[피칭 스크립트 붙여넣기]
```

→ 예상 질문에 답해보고, 다듬을 부분 찾기
