## EXPLAIN

### 자동화 아이디어 구체화

Session 1-2에서 말했던 "자동화하고 싶은 업무"를 구체적으로 설계한다.

**좋은 자동화 아이디어의 조건**:
1. **반복적**: 매일/매주 하는 일
2. **패턴이 있음**: 항상 비슷한 형태로 처리됨
3. **시작점이 명확**: 어떤 입력을 받아서 어떤 결과를 내는지 알 수 있음

**LiveKlass 추천 자동화 아이디어 목록**:

| 아이디어 | 입력 | 출력 |
|---------|------|------|
| CS 응답 초안 생성 | 고객 문의 메일 | 응답 초안 |
| 파이프라인 주간 리포트 | 구글시트 파이프라인 | 주간 요약 + 액션 |
| 아침 Slack 브리핑 | Slack 채널들 | 오늘의 핵심 정보 |
| 미팅 준비 체크리스트 | 고객사 이름 | 미팅 전 확인사항 |
| 월간 VoC 분석 | 구글시트 VoC | 카테고리별 트렌드 |

### SKILL.md 설계 템플릿

```markdown
---
name: [skill-name]
description: [한 줄 설명. 이 skill이 어떤 상황에서 사용되는지]
---

# [스킬 이름]

## 목적
[이 스킬이 해결하는 문제]

## 입력
[어떤 정보가 필요한가]

## 처리 절차
[Claude가 어떤 순서로 처리하는가]

## 출력
[어떤 형태로 결과를 보여주는가]
```

## EXECUTE

지금 바로 나만의 자동화 아이디어를 설계해보세요:

Claude Code에 입력:
```
나는 라이브클래스 [직군]이야.
매일/매주 반복하는 업무 중에서 Claude로 자동화하면 좋을 것 3가지를 찾아줘.
각각의 입력-처리-출력 구조도 설계해줘.
```

→ 결과를 보고 가장 마음에 드는 것 하나를 Block 1에서 직접 만든다.

## QUIZ

AskUserQuestion으로 물어본다:

```json
AskUserQuestion({
  "questions": [{
    "question": "어떤 자동화를 만들어볼 건가요? 아이디어를 하나 골라주세요.",
    "header": "자동화 아이디어 선택",
    "options": [
      {"label": "위 추천 목록에서 선택", "description": "CS 응답, 파이프라인 리포트, Slack 브리핑 등"},
      {"label": "직접 아이디어 입력", "description": "나만의 자동화 아이디어가 있어요"}
    ],
    "multiSelect": false
  }]
})
```

답변을 바탕으로:
- 입력/출력이 명확한지 확인
- 너무 복잡하면 "이것부터 시작하는 건 어때요?" 제안
- 좋은 아이디어면 "완벽해요! Block 1에서 바로 만들어봅시다"
