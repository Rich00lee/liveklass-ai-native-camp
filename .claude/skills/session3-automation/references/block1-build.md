## EXPLAIN

### Skill 제작: SKILL.md 작성

Block 0에서 설계한 아이디어를 실제 SKILL.md 파일로 만든다.

**파일 위치**:
```
~/.claude/skills/[skill-name]/SKILL.md
```

**작성 원칙**:
1. **명확한 트리거**: 언제 이 Skill을 쓸지 description에 적기
2. **단계별 절차**: Claude가 순서대로 따라할 수 있게 구체적으로
3. **출력 형식 지정**: "3줄 요약", "테이블 형식" 등 원하는 형태 명시
4. **예시 포함**: 입력 예시 / 출력 예시를 넣으면 정확도 올라감

**예시: CS 응답 초안 스킬**

```markdown
---
name: cs-reply
description: 고객 CS 문의 이메일을 받아 응답 초안을 생성한다. "CS 응답", "고객 문의", "답변 초안" 요청에 사용.
---

# CS 응답 초안 생성

## 역할
나는 라이브클래스 CS 담당자다.
고객의 문의를 받아 공손하고 명확한 응답 초안을 작성한다.

## 처리 절차
1. 고객 문의 핵심 파악
2. 라이브클래스 정책 기준으로 답변 방향 결정
3. 응답 초안 작성 (200자 내외)
4. 추가 확인이 필요한 사항 표시

## 출력 형식
- 제목: "Re: [원본 제목]"
- 인사말 → 핵심 답변 → 추가 안내 → 마무리 인사
- 끝에 "[확인 필요: OOO]" 항목 별도 표시

## 주의사항
- 환불 정책: 결제 후 7일 이내, 진도율 20% 미만
- 불확실한 경우: 명확히 확인 후 답변한다고 안내
```

## EXECUTE

Claude Code에 아래처럼 요청하면, Claude가 직접 파일을 만들어줍니다:

```bash
# Claude Code에 입력:
.claude/skills/[내가 정한 skill 이름]/SKILL.md 파일을 만들어줘.

아래 내용을 기반으로 작성해줘:
- 목적: [내 자동화 아이디어]
- 입력: [어떤 정보를 받나]
- 출력: [어떤 형태로 결과를 보여줄까]
```

→ Claude가 직접 스킬 파일을 생성합니다. 사용자가 직접 파일을 만들 필요 없이, Claude Code에 요청하면 됩니다.

파일이 만들어지면 `/[skill-name]`으로 바로 실행해보세요!

## QUIZ

AskUserQuestion으로 물어본다:

질문: "SKILL.md 파일이 만들어졌나요? 실행해봤을 때 결과가 어떤가요?"
- "잘 됐다" → ✅ "완성이에요! 이제 이 Skill은 팀 전체에 공유할 수 있어요."
- "결과가 이상하다" → "어떤 부분이 이상한가요? SKILL.md의 [처리 절차] 또는 [출력 형식]을 수정해봅시다."
- "실행이 안 된다" → "파일 경로 확인: `~/.claude/skills/[이름]/SKILL.md` 이렇게 되어 있나요?"
