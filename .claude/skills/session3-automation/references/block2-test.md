## EXPLAIN

### 테스트 & 개선

만든 Skill을 검증하고 다듬는다. "동작하는 결과물"에서 "믿을 수 있는 결과물"로.

**3가지 테스트 시나리오**:

1. **정상 케이스**: 일반적인 입력을 넣었을 때 잘 동작하는가?
2. **극단 케이스**: 정보가 너무 적거나 너무 많을 때?
3. **예외 케이스**: 예상치 못한 입력이 들어왔을 때?

**개선 포인트 체크리스트**:
- [ ] 출력 형식이 실무에서 바로 쓸 수 있는 형태인가?
- [ ] 처리 시간이 너무 길지 않은가? (30초 이내 권장)
- [ ] 결과가 일관성 있는가? (같은 입력 → 비슷한 출력)
- [ ] CLAUDE.md와 충돌하지 않는가?

**팀 공유 준비**:
Skill 파일을 팀원에게 공유하려면:
```bash
# 방법 1: 파일 직접 공유
# ~/.claude/skills/[skill-name]/SKILL.md 파일을 Slack/Notion으로 공유

# 방법 2: GitHub에 올리기 (Session 4 후속 과제)
# npx skills add [github-username]/[repo-name] --skill [skill-name]
```

## EXECUTE

3단계 테스트를 해보세요:

**Step 1 - 정상 테스트**:
```
/[내 skill 이름]
[실제 업무에서 쓸 법한 예시 데이터 입력]
```

**Step 2 - 개선**:
결과가 마음에 안 드는 부분을 SKILL.md에서 수정:
```
SKILL.md에서 [어떤 부분]을 수정해줘.
[원하는 방향으로] 바꿔줘.
```

**Step 3 - 공유 준비**:
```
이 Skill을 팀원에게 설명하는 사용 가이드를 SKILL.md 하단에 추가해줘.
비개발자도 바로 쓸 수 있도록 간단하게 작성해줘.
```
