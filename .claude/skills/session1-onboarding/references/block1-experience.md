## EXPLAIN

### Working Backward — 완성품부터 보기

Claude Code가 뭘 할 수 있는지, 결과물부터 먼저 보자.
LiveKlass 실무에서 실제로 쓰이는 3가지 예시를 데모로 보여준다.

---

**데모 1: CS 응답 초안 (30초)**

상황: 고객이 "수강권 환불 가능한가요?" 문의 메일 보냄

Claude Code에 이렇게 입력:
```
고객 문의: "결제한 지 5일 됐는데 환불 가능한가요? 강의가 기대와 달라서요."

라이브클래스 환불 정책 기준으로 공손하고 명확한 답변 초안 작성해줘.
```
→ 30초 안에 완성된 CS 응답 초안이 나온다

---

**데모 2: 구글시트 데이터 분석 (1분)**

상황: 월간 세일즈 파이프라인 데이터를 분석하고 싶음

Claude Code에 이렇게 입력:
```
파이프라인 데이터 분석해줘:
- A사: 제안 단계, 예상 금액 500만원, D+14일
- B사: 계약 단계, 예상 금액 1200만원, D+3일
- C사: 미팅 단계, 예상 금액 300만원, D+30일

이번 달 예상 매출과 우선순위 액션 알려줘.
```
→ 분석 결과 + 우선순위 액션 플랜이 나온다

---

**데모 3: Slack 메시지 요약 (1분)**

상황: 오전에 Slack 메시지가 너무 많아서 중요한 것만 정리하고 싶음

Claude Code에 이렇게 입력:
```
오늘 아침 Slack 메시지를 요약해줘. 액션 아이템, 결정 사항, 단순 공유 세 카테고리로 분류해서 정리해줘.

[메시지 붙여넣기]
```
→ 깔끔하게 정리된 요약본이 나온다

## EXECUTE

지금 직접 3가지 중 **하나를 선택**해서 실행해보세요:

1. **CS 응답 초안**: 실제 받은 문의 메일 하나 붙여넣고 응답 초안 요청
2. **데이터 분석**: 지금 갖고 있는 파이프라인 데이터 붙여넣고 분석 요청
3. **Slack 요약**: 오늘 받은 Slack 메시지 일부 붙여넣고 요약 요청

> 💡 아직 데이터가 없으면, "예시 데이터 만들어줘"라고 Claude에게 부탁하면 됩니다.

## QUIZ

AskUserQuestion으로 소감을 물어본다:

질문 1: "데모를 직접 실행해봤을 때, 가장 인상 깊었던 점은?"
- "빠르기" → "맞아요! 보통 30초~2분이면 됩니다. 반복 업무의 80%를 이 속도로 처리할 수 있어요."
- "정확도" → "기대 이상이죠? CLAUDE.md에 컨텍스트를 넣으면 더 정확해집니다 — Block 2에서 배울 거예요."
- "활용 범위" → "세 가지는 맛보기예요. 오늘 세션 끝나면 자신의 업무에 맞게 커스터마이징할 수 있어요."
- 기타 → 공감 + Block 2 예고

질문 2: "오늘 가장 자동화하고 싶은 업무 하나를 말해볼까요? (나중에 Session 3에서 직접 만들 거예요)"
→ 답변을 기억해두었다가 Session 3에서 활용한다고 알려준다
