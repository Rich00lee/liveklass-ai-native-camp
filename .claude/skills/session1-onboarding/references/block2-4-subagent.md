## EXPLAIN

### Subagent — 여러 AI가 동시에 일한다

Subagent는 Claude가 **여러 개의 AI 작업자를 동시에 띄워** 병렬로 일을 처리하는 기능이다.

**비유**: 혼자 김밥 10줄 마는 것 vs 10명이 동시에 1줄씩 마는 것
→ Subagent = 10명이 동시에 작업하는 구조

**LiveKlass 활용 예시**:
```
"10개 고객사의 최근 문의를 분석해줘."
```
→ Subagent 없이: 10개를 순서대로 하나씩 분석 (느림)
→ Subagent 있이: 10개를 동시에 분석 (10배 빠름)

**실무에서 유용한 상황**:
- 여러 고객사 데이터를 동시에 분석할 때
- 동시에 여러 파일을 처리할 때
- 복잡한 리서치 작업을 여러 방향에서 동시 진행할 때

지금 이 워크숍에서도 Subagent가 활용된다:
→ 진행자가 없어도 AI가 설명-실습-퀴즈를 자동으로 진행하는 구조

## EXECUTE

Subagent를 간단히 체험해보세요:

Claude Code에 입력:
```
Task 도구를 써서 다음 3가지를 동시에 분석해줘:
1. 오늘 내가 처리해야 할 최우선 업무 예측
2. 이번 주 가장 중요한 세일즈 액션
3. 팀에 공유해야 할 정보 항목
```

→ Claude가 여러 방향으로 동시에 생각하는 걸 확인해보세요.

## QUIZ

AskUserQuestion으로 퀴즈 출제:

질문: "Subagent를 쓰면 가장 효과적인 업무는 무엇일까요?"
- "단순 반복 작업 (예: 10개 파일 동시 처리)" → ✅ 정답! "독립적인 작업을 병렬로 처리할 때 효과가 극대화돼요."
- "순서가 있는 작업 (예: A 결과를 보고 B를 결정)" → "그런 경우는 순서대로 처리해야 해요. Subagent는 독립적인 작업에 적합해요."
- "대화 중 실시간 응답" → "대화는 Subagent보다 일반 대화가 더 빨라요. Subagent는 복잡한 작업 처리에 강점이 있어요."
